# DCE 5.0 v0.36.0-20251130 Enterprise Released

This page explains the latest features, improvements and bug fixes, included
in DCE 5.0 v0.36.0-20251130 Enterprise. DCE 5.0 modularizes each product module,
and the versions included in this release are the default versions
in the DCE 5.0 installer. If you need to independently upgrade or replace each module, contact
Daocloud's support team to download the proper versions and upgrade. Also refer to
[DCE 5.0 Enterprise Deployment Requirements](../../install/commercial/deploy-requirements.md).

## Upgrade Notes

- Microservices Engine
    
    In this release, the CRDs of Microservice Engine v0.51.0 have been updated.  
    Before upgrading, run the following commands on your working cluster to manually install the CRDs for this version:

    ```shell
    # projectcontour-related CRDs
    kubectl apply -f skoala-init/charts/contour-provisioner/crds/contour.yaml
    kubectl apply -f skoala-init/charts/contour-provisioner/crds/higress.gen.yaml
    kubectl apply -f skoala-init/charts/contour-provisioner/crds/istio-envoyfilter.yaml

    # gateway-api-related CRDs
    kubectl apply -f skoala-init/charts/contour-provisioner-prereq/crds/gateway-api.yaml
    ```

- Insight

    Insight has deprecated [Jaeger v1](https://github.com/jaegertracing/jaeger/issues/6321), migrated to Jaeger v2,
    and incorporated the Jaeger v2 collector into the Global OpenTelemetry Collector as a plugin.
    Before upgrading from DCE 5.0 v0.32.0 to v0.36.0, add the configuration item `jaeger.collector.enabled: false` under the
    `components.insight.variables` field in the manifest.yaml.
