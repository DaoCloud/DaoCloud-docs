# DCE 5.0 v0.35.0-20251031 Enterprise Released

This page explains the latest features, improvements and bug fixes, included
in DCE 5.0 v0.35.0-20151031 Enterprise. DCE 5.0 modularizes each product module,
and the versions included in this release are the default versions
in the DCE 5.0 installer. If you need to independently upgrade or replace each module, contact
Daocloud's support team to download the proper versions and upgrade. Also refer to
[DCE 5.0 Enterprise Deployment Requirements](../../install/commercial/deploy-requirements.md).

## Upgrade Notes

- AI Lab v0.22.0 upgrades Kueue to v0.14.1. Since the CRDs have changed, the worker clusters must manually upgrade the baize-agent. Please follow the steps below:

    1. Run the following command to manually delete the old CRDs:

        Delete the `topologies.kueue.x-k8s.io` and `cohorts.kueue.x-k8s.io` CRD resources.

        ```shell
        kubectl delete crd topologies.kueue.x-k8s.io && kubectl delete crd cohorts.kueue.x-k8s.io
        ```

    2. Upgrade the baize-agent:

        Go to **Container Management** -> **Helm Apps** and perform the baize-agent upgrade, or directly run `helm upgrade` in the command line to complete the upgrade.

- Insight

    Insight has deprecated [Jaeger v1](https://github.com/jaegertracing/jaeger/issues/6321), migrated to Jaeger v2,
    and incorporated the Jaeger v2 collector into the Global OpenTelemetry Collector as a plugin.
    Before upgrading from DCE 5.0 v0.32.0 to v0.35.0, add the configuration item `jaeger.collector.enabled: false` under the
    `components.insight.variables` field in the manifest.yaml.
