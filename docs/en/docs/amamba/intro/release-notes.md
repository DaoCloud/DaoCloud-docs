# Releases Notes

This page lists the release notes for Workbench to help you understand
the development and feature changes in each version. 

## 2025-11-30

### v0.39.0

- **Added** support for enabling and disabling pipelines
- **Added** an API to add or modify pipeline parameters
- **Fixed** an issue where DSN did not support timezone configuration
- **Fixed** an issue where traces were lost during Jenkins calls
- **Improved** remove redundant Jenkins calls

## 2025-10-31

### v0.38.1

- **Fixed** an issue where the global build parameter type was incorrect.

## 2025-08-31

### v0.38.0

- **Added** support for deleting deprecated toolchain resources.
- **Added** support for passing upstream pipeline information when running pipelines via OpenAPI and Webhooks.
- **Added** support for the Jenkins `Extended Choice Parameter` and `Extensible Choice Parameter` plugins.
- **Added** redesigned pipeline metrics and dashboards.
- **Fixed** an issue where creating the control-plane namespace for a GitOps module failed.
- **Fixed** an issue where pipeline runs did not preserve the names of uploaded files.
- **Fixed** an issue where the schedule configuration for periodic toolchain synchronization was incorrect.
- **Fixed** an issue where Jenkins credentials were not updated when updating the `sonarqube` configuration.
- **Fixed** an issue where unique index conflicts occurred during toolchain project synchronization.
- **Improved** ArgoCD links to reduce unnecessary token generation.
- **Improved** pipeline templates generated by `from-git` and `from-jar`.

## 2025-06-30

### v0.37.0

- **Added** support for release statistics.
- **Added** support for displaying pipeline build parameters in the run history list.
- **Added** support for fuzzy search in release records.
- **Added** support for adding and removing existing resources in native applications.
- **Added** support for editing native applications via YAML.
- **Added** support for modifying integrated toolchain configurations.
- **Added** support for parameter validation when running pipelines.
- **Fixed** an issue where `Rollout` always failed after enabling monitoring analysis.
- **Fixed** an issue where unnecessary `Gateway` resources were created. Impact: reinstalling Ghippo after enabling the `Argo CD UI` caused the DCE 5 UI to become inaccessible.
- **Fixed** an issue where copying a pipeline caused Webhooks to be triggered repeatedly.
- **Fixed** an issue where a newly copied pipeline showed an incorrect execution status in error records.
- **Fixed** an issue where user group permissions were not handled correctly during pipeline approval.
- **Fixed** an issue where build parameters defined in a Jenkinsfile were not triggered.
- **Fixed** an issue where aliases and descriptions did not take effect when updating an application.
- **Improved** the ArgoCD connection mechanism to improve stability.
- **Improved** the way password-type pipeline parameters are passed to enhance security.
- **Improved** the synchronization logic for ArgoCD `destination`, `project`, and `cluster` to make workflows more robust.

## 2025-04-30

### v0.36.0

- **Added** description and executor fields to pipeline run detail view.
- **Fixed** 403 error when fetching the `Nacos` instance list.

## 2025-02-28  

### v0.35.0  

- **Added** pipeline templates for deploying to virtual machines.  
- **Added** two new `FeatureGates`: `ArgoCDUI` and `AppAnyNamespace`.  
- **Added** support for modifying pipeline cache size, controlled by the `AllowVolumeExpansion` field.  
- **Fixed** the namespace selector in the `ArgoRollouts` monitoring dashboard not filtering by cluster.  
- **Fixed** display issues with `gitops event`.  
- **Fixed** the creation of `Application` resources in *app of apps* mode to ensure they are placed in the `argocd` namespace.  
- **Fixed** missing cluster and namespace information in the default integrated `jenkins` instance.  
- **Fixed** confusing logic in namespace display.  
- **Fixed** an issue where updating container images in `rollouts` would unintentionally modify other fields.  

!!! warning  

    When manually upgrading the `amamba` component using a **Helm app**, the `jenkins` image registry address will remain `xxx/docker.m.daocloud.io`.  
    When upgrading via the installer, it will automatically update to `xxx/ghcr.m.daocloud.io`.
