# Hot Topics

This page lists some hot topics in the cloud native community to help everyone understand trends and directions. For more, see [OSS Insight](https://ossinsight.io/).

- [First Dual-Stack Istio Solution Implementation Experience in China, Implementation Code Open Source](https://mp.weixin.qq.com/s/E9HZIkSbZ3BLetDL0ZhzvA)

    In widely used service mesh projects (Istio and Linkerd as examples), dual-stack technology support is still missing.
    To better achieve coordination between service mesh and Kubernetes in dual-stack support,
    this article mainly introduces Istio's implementation scheme in dual-stack technology and its implementation scenarios in China Mobile Cloud.
    However, this scheme is just an [experimental branch](https://github.com/istio/istio/tree/experimental-dual-stack) and does not have complete unit tests and integration tests.

- [Where Should Microservices Go After Entering the Deep Water Zone?](https://mp.weixin.qq.com/s/yBY-E-tndUJCmA4KYRfrDw)

    After microservices enter the deep water zone, developers and architects care more about microservice security, stability, cost optimization, microservice governance standardization, and promoting cloud native microservice architecture to evolve into multi-runtime microservice architecture.
    For the limitations of multi-runtime architecture, more standardized and platformized service mesh development and operation capabilities need to be implemented, standardization of Sidecar and runtime definitions, while making the operation platform more standard and easy to use.

- [2023 Cloud Native Development Trend Predictions](https://mp.weixin.qq.com/s/QePkownt0_Ex9RWWeGtaag)

    This article is CNCF CTO Chris Aniszczyk's predictions about hot topics in cloud native and technology areas in 2023,
    including: cloud native IDE becomes the norm, FinOps becomes mainstream and shifts left, open source SBOM everywhere, GitOps matures and enters peak production, OpenTelemetry matures, Backstage developer portal continues to improve, WebAssembly enters steady climb period, etc.
