# DCE 5.0 v0.35.0-20251031 商业版正式发布

本页说明了 DCE 5.0 v0.35.0-20251031 商业版包含的最新特性、优化、缺陷修复等。
DCE 5.0 每个产品模块都是独立解耦的，本次发布中所涉及的产品模块为 DCE 5.0 安装器默认包含的版本。
如需独立升级或替换各个模块，请联系 Daocloud 售后支持人员下载对应版本并升级。
另请参阅[商业版部署要求](../../install/commercial/deploy-requirements.md)。

## 升级注意事项 <a id="upgrade-notes" />

- AI Lab v0.22.0 升级 Kueue 至 v0.14.1，由于 CRD 发生变更，工作集群需手动升级 baize-agent，请按照以下步骤操作：

    1. 执行如下命令，手动删除旧版本 CRD

        删除 `topologies.kueue.x-k8s.io` 和 `cohorts.kueue.x-k8s.io` CRD 资源。

        ```shell
        kubectl delete crd topologies.kueue.x-k8s.io && kubectl delete crd cohorts.kueue.x-k8s.io
        ```

    2. 升级 baize-agent

        进入 **容器管理** -> **Helm 应用** 页面，执行 baize-agent 升级操作，或者直接通过命令行执行 `helm upgrade` 完成升级。

- 可观测性 Insight

    Insight 废弃 [Jaeger v1](https://github.com/jaegertracing/jaeger/issues/6321) ，迁移至 v2 并将 Jaeger v2 collector
    作为插件合并进了 Global OpenTelemetry Collector。 从 DCE 5.0 v0.32.0 升级到 v0.35.0 之前，请在产品清单文件 manifest.yaml 中的`components.insight.variables`
    下添加配置：`jaeger.collector.enabled: false`。
